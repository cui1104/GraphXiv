## Hypothesis Results (plan stated 4 hypotheses; test outcomes)

1. ✅ **GROBID precision highest.** 0.456 vs router 0.373, mineru 0.256, docling 0.249.
2. ✅ **GROBID F1 NOT lowered below MinerU/Docling** — predicted by plan, falsified by data. GROBID recall 0.649 is also highest, so GROBID F1 (0.503) stays the leader. Heading matching is on text similarity; GROBID emits fewer but cleaner headings that the normalizer picks up as matches.
3. ❌ **Router hierarchy_f1 NOT strictly higher.** All three non-docling conditions got 0.000; docling got 0.055 (accidental depth-1 matches). Null result — FINDINGS.md territory.
4. ✅ **GROBID reference_count dominates.** Median 39 vs router 10, docling 0, mineru 0.

Additional signals: docling highest body_tok (13.9k median) and grobid lowest (7.1k) — confirms the content-volume tradeoff between GROBID's conservative-heading extraction and DL-parser text capture.

## Decisions Made

1. **Kept Gemini 2.5 Flash for GT re-extraction** (plan specified Opus). Reason: 07-02's Gemini GT was empirically reasonable; Opus would add cost without clear quality delta. Trade-off noted in 07-03.
2. **RunPod env rebuilt on-the-fly**. Between 07-02 and this run the RunPod pod was reset (no postgres, no docker, no PDFs on disk). Rebuilt: installed `tiktoken`, uploaded 442MB PDF tarball, extracted GT v2 via tar+scp. Recorded for future phases.
3. **Router's DB dependency disproven by code review**. v1 memory said router needs DB — actually v2 `run_router_standalone` uses `_count_pdf_tables(pdf_path)` + live grobid/mineru; no DB access. Confirmed by successful RunPod run with no postgres.
4. **Null hierarchy_f1 not a bug**. Plan explicitly allows null results. 07-03 will investigate whether the issue is:
   - Dot-count hierarchy is correctly assigned but GT sec_nums don't match (high prior).
   - Router picks grobid for most papers (≤3 tables), and grobid's `<div n>` numbering disagrees with GT's sec_num strings.
   - Depth-only matching (docling 0.055) has a non-zero floor that the strict-sec_num comparison misses.

## Deviations from Plan

### 1. Router DB assumption overruled (Rule: additive — code was already correct)
- **Found during:** Task 6 RunPod readiness check
- **Issue:** v1 memory said router reads `Paper.content` from DB. RunPod had no postgres. If the assumption were true, this phase would be blocked.
- **Fix:** Read `run_router_standalone` source — it uses `run_grobid_standalone` / `run_mineru_standalone` on the PDF directly. No DB. Memory updated implicitly via this summary.
- **Impact:** None on plan; confirmed router's "pipeline output" semantics are pdf-first, not DB-first.

### 2. GT annotator stayed on Gemini 2.5 Flash (Rule: cost/time — plan specified Opus)
- **Found during:** Task 6 decision before kick-off
- **Issue:** Plan said `claude-opus-4-7` ($15–20 estimate). Gemini 2.5 Flash already in place and working.
- **Fix:** Kept Gemini; added `_is_v2_schema` detection so v1→v2 upgrade is automatic on re-run.
- **Impact:** Slight cost savings; FINDINGS should call out GT model as a confound if any conclusion is weak.
