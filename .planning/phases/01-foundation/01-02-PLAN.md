Create a **hand-written** Alembic migration (NOT autogenerated). This avoids the GIN index autogenerate false-positive bug (Alembic issue #1390).

**Migration file: `alembic/versions/0001_initial_schema.py`**

Revision ID: use a fixed hex string like `0001abcdef01` (or generate with `alembic revision`).

The `upgrade()` function must do the following IN THIS ORDER:

1. **Enable pgvector extension** (MUST be before any table using vector type):
```python
op.execute("CREATE EXTENSION IF NOT EXISTS vector")
```

2. **Create `papers` table** using `op.create_table()`:
   - All columns from the Paper model (canonical_id UUID PK, arxiv_id TEXT UNIQUE, pmc_id TEXT UNIQUE, doi TEXT, title TEXT, abstract TEXT, year INTEGER, venue TEXT, parse_source TEXT, parse_quality TEXT, token_count INTEGER, tldr TEXT, embeddings vector(768), content JSONB, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now())
   - For the vector column, use `sa.Column("embeddings", Vector(768), nullable=True)` — import Vector from pgvector.sqlalchemy

3. **Create `paper_sources` table**:
   - id SERIAL PK, canonical_id UUID FK(papers.canonical_id), source_type TEXT, asset_path TEXT, parse_status TEXT DEFAULT 'pending', created_at TIMESTAMPTZ DEFAULT now()

4. **Create `id_map` table**:
   - id SERIAL PK, arxiv_id TEXT, pmc_id TEXT, doi TEXT, canonical_id UUID FK(papers.canonical_id)

5. **Create `crawl_state` table**:
   - id SERIAL PK, source TEXT NOT NULL, resumption_token TEXT, last_harvested_at TIMESTAMPTZ, record_count INTEGER DEFAULT 0

6. **Create `paper_citations` table** (INFRA-06):
   - id SERIAL PK, source_paper_id UUID FK(papers.canonical_id) NOT NULL, target_paper_id UUID FK(papers.canonical_id) NULLABLE, target_arxiv_id TEXT, target_doi TEXT, context_text TEXT

7. **Create indexes** (after all tables exist):
   - `op.create_index("idx_paper_citations_source", "paper_citations", ["source_paper_id"])`
   - `op.create_index("idx_paper_citations_target", "paper_citations", ["target_paper_id"])`
   - `op.create_index("idx_papers_year", "papers", ["year"])`

8. **GIN FTS index as raw SQL** (per RESEARCH.md Pattern 5):
```python
op.execute("""
    CREATE INDEX idx_papers_fts
    ON papers
    USING GIN (to_tsvector('english', coalesce(title, '') || ' ' || coalesce(abstract, '')))
""")
```

The `downgrade()` function drops all tables in reverse order and drops the extension:
```python
op.drop_table("paper_citations")
op.drop_table("crawl_state")
op.drop_table("id_map")
op.drop_table("paper_sources")
op.drop_table("papers")
op.execute("DROP EXTENSION IF EXISTS vector")
```
