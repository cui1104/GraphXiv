IMPORTANT: This task supersedes Plan 02's sync `def` handler pattern. It converts all route handlers from sync `def` to `async def` to enable async Redis cache calls. Per D-23, sync `SessionLocal` is preserved for all DB calls -- they are wrapped in `asyncio.to_thread()` so the event loop stays free. No asyncpg or AsyncSession is introduced.

Since route handlers from Plan 02 are sync `def` (not `async def`), they cannot directly call `await redis.get()`. Two approaches:
- (A) Change handlers to `async def` and use `asyncio.to_thread` for DB calls
- (B) Use sync `redis.Redis` alongside `redis.asyncio.Redis`

**Recommended approach (A):** Convert route handlers to `async def` and wrap sync DB calls in `asyncio.to_thread()`. This lets us use the async Redis client from `get_redis` directly. The DB work is already the heaviest part; wrapping it in `to_thread` keeps the event loop free.

1. **Create cache utility module** `app/api/cache.py`:
   ```python
   import json
   import hashlib
   from redis.asyncio import Redis
   
   PAPER_TTL = 3600   # 1 hour (D-17)
   SEARCH_TTL = 300   # 5 minutes (D-17)
   
   async def get_cached(redis: Redis, key: str) -> dict | None:
       cached = await redis.get(key)
       if cached:
           return json.loads(cached)
       return None
   
   async def set_cache(redis: Redis, key: str, data: dict, ttl: int) -> None:
       await redis.set(key, json.dumps(data, default=str), ex=ttl)
   
   def paper_cache_key(canonical_id: str, view: str) -> str:
       return f"papers:{canonical_id}:{view}"
   
   def search_cache_key(q: str, limit: int, search_mode: str) -> str:
       raw = f"{q}:{limit}:{search_mode}"
       md5 = hashlib.md5(raw.encode()).hexdigest()
       return f"search:{md5}"
   ```

   Add the invalidation function:
   ```python
   def _invalidate_cache(paper) -> None:
       """Delete all cached views for this paper from Redis.
       
       Uses sync redis.Redis (NOT redis.asyncio) because Celery tasks
       run synchronously (Pitfall 6 from RESEARCH.md).
       """
       import redis as redis_lib
       from app.config import get_settings
       settings = get_settings()
       r = redis_lib.from_url(settings.redis_url)
       try:
           pattern = f"papers:{paper.canonical_id}:*"
           cursor = 0
           while True:
               cursor, keys = r.scan(cursor, match=pattern, count=100)
               if keys:
                   r.delete(*keys)
               if cursor == 0:
                   break
       finally:
           r.close()
   ```
   
   Use `SCAN` (not `KEYS`) per anti-pattern guidance in RESEARCH.md. Create a fresh sync Redis connection per invocation (lightweight for the low frequency of normalize calls). Wrap in try/except so cache invalidation failure does not break the normalize task:
   ```python
   try:
       _invalidate_cache(paper)
   except Exception as exc:
       logger.warning("Cache invalidation failed for %s: %s", paper.canonical_id, exc)
   ```
